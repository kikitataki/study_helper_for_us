# 授業内容共有サイト

動画url:https://www.youtube.com/watch?v=ZpZEmvBaJUo

授業ごとの要約・コメント・補足情報を共有できるWebアプリです。
授業名と講義回数を選択すると、登録されている講義ノートを閲覧できます。
また、ログイン済みユーザーはコメント投稿・返信・お気に入り登録・通知確認・お問い合わせ送信などを行えます。

## 概要

このアプリは、授業内容をあとから振り返りやすくすることを目的として作成しました。

授業を欠席した人や、講義中に内容を十分に整理できなかった人でも、授業ごとの要約や補足コメントを確認できるようにしています。
また、コメントや返信機能によって、利用者同士で補足説明や疑問点の共有ができるようにしました。

## 主な機能

### 録音機能

授業内容を録音し、講義ノートとして投稿できます。

※録音機能を利用する際は、授業担当者の許可や大学のルールを確認した上でご利用ください。

### ログイン・新規登録申請

Firebase Authenticationを用いたログイン機能を実装しています。
新規ユーザーは登録申請を行い、管理者が承認することで利用可能になります。

### 授業ノート閲覧

授業名と講義回数を選択すると、Firestoreに保存された授業要約を表示します。
要約文には独自の記法を用いて、見出し・強調・箇条書きなどを整形して表示します。

### 授業内容投稿

ログイン済みユーザーは、授業名・講義回数・授業内容を入力して投稿できます。
本番環境では入力内容をそのまま保存し、ローカル環境では必要に応じてAIによる講義ノート化を行える設計にしています。

### コメント・返信機能

各授業ページにコメントを投稿できます。
また、コメントに対して返信できるようにし、簡単な木構造の掲示板として利用できるようにしました。

### お気に入り機能

よく閲覧する授業をお気に入り登録できます。
お気に入り登録した授業に関する新しい投稿やコメントを通知対象にしています。

### アプリ内通知機能

お気に入り登録している授業に新しい要約やコメントが投稿された場合、アプリ内通知として通知されます。
通知をクリックすると、該当する授業名・講義回数のページへ直接遷移できます。

### お問い合わせ・改善案

ユーザーはお問い合わせや改善案を送信できます。
管理者は投稿されたお問い合わせ内容を確認できます。

### 管理者機能

管理者は以下の操作を行えます。

* 新規登録申請の確認
* ユーザー承認
* 登録ユーザー管理
* 授業ページ削除
* コメント削除
* お問い合わせ確認

## 工夫した点

### Reactによるコンポーネント分割

旧HTML中心の実装からReactへ移行し、画面や機能ごとにコンポーネントを分割しました。
MVCの考え方を参考に、画面表示をcomponents/pages、データ取得・保存処理をservices、共通の状態管理や認証処理をhooksに分離し、責務が明確になるようにReact化しました。

### Firebaseとの連携

ログイン状態、承認済みユーザー判定、授業データ、コメント、通知、お気に入りなどをFirebaseで管理しています。
Firestore Rulesも利用し、管理者・承認済みユーザー・未ログインユーザーで操作権限を分けています。

### 通知機能

全授業ではなく、お気に入り登録した授業だけを通知対象にしました。
これにより、ユーザーに関係のない通知が増えすぎないようにしています。

### コメントの返信構造

コメントに対して返信できるようにし、授業ごとの補足や質問のやり取りをしやすくしました。
Firestoreのサブコレクションを用いて、コメントと返信を階層的に管理しています。

### スマホ対応

スマホからも使いやすいように、入力欄の文字サイズやレイアウトを調整しました。
また、ホーム画面に追加した際のアイコン表示にも対応しています。

## 開発背景

授業内容をあとから復習したいときに、ノートが不十分だったり、欠席して内容が分からなかったりすることがあります。
そこで、授業ごとに要約や補足情報を共有できるアプリを作成しました。

単に要約を見るだけでなく、利用者同士がコメントや返信で情報を補足できるようにすることで、授業理解を助ける場として使えることを目指しています。

## 現在の課題

### 録音機能のフロントエンド移行

現在、録音に関する処理の一部はバックエンド側で実装しています。
しかし、録音は本来ユーザーのブラウザ上で行われる処理であり、React側で完結させる方が自然です。

### フロントエンドとバックエンドの二重サーバー構成

現在の開発環境では、Reactの開発サーバーとFlaskのバックエンドサーバーを別々に起動する必要があります。

```txt
frontend: Vite / React
backend: Flask
```

そのため、開発時には2つのWebサーバーを同時に立ち上げる必要があり、環境構築や動作確認がやや複雑になっています。
特に、ローカル環境と本番環境の違いを意識する必要があります。

今後は、バックエンドに依存している処理を整理し、可能な処理はフロントエンドへ移行する予定です。
また、AI要約などサーバー側で安全に実行すべき処理については、Firebase Functionsなどへの移行も検討しています。


## 利用上の注意

本アプリは学習補助を目的とした授業メモ・要約共有アプリです。
授業の録音や資料の利用については、大学・授業担当者の方針、著作権、プライバシーに関わる可能性があります。
そのため、録音機能を使用する場合は、事前に許可や利用ルールを確認してください。
また、録音データや授業資料そのものの無断共有は想定しておらず、共有対象は学習目的で整理した要約・コメントに限定する設計としています。
